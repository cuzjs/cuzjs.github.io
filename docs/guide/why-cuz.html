为什么使用 cuz?

面对目前日益繁杂的前端开发, cuz 的目标是简化开发流程, 整合开源社区的高可用组件, 使其能通过统一的方式进行调用. 让大家将注意力从复杂的 DOM 交互转移到实际业务中.

cuz 主要分为以下几个部分:

  • cuz: 模块化构建框架核心, 包括第三方依赖, 如 React, Redux.
  • cuz-cli: 辅助开发命令行工具, 基于 NodeJS. 用于快速的进行调试, 构建及发布.
  • cuz-ui: React 组件库, 基于 Bootstrap.
  • cuz-themes: CSS 主题, 支持多主题切换.
  • cuz-charts: 图表库, 基于 EChart. 实现仅提供 WEB API 和简单配置即可生成图表的能力.
