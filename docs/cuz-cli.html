cuz-cli

cuz-cli 是 cuz 的命令行接口工具, 基于 Webpack 实现, 用于对项目进行调试和构建.

特性 #

  • 支持 ES6 & ES7
  • 支持 LESS & SASS
  • 集成 ESLint 语法检查
  • 集成 Mocha 单元测试套件
  • 支持构件时声明不同的环境
  • 支持打包静态资源文件
  • 支持配置 CDN 发布路径

使用说明 #

安装 #

$ npm install cuz-cli -g

Note:

命令行工具安装完成后, 需安装具体 cuz 版本才可使用下面的命令.

使用 #

初始化项目 #

初始化命令可以生成一个标准模版项目, 您可以直接模版上进行后续开发.

$ cuz init

本地调试 #

本地调试命令将启动一个 localhost 的 Web 环境供浏览器直接访问, 默认端口号为 5000, 您也可以在 配置文件 中修改它以避免冲突.

$ cuz dev

构建 #

构建命令将自动打包项目中使用到的所有静态资源文件, 并生成到 build 目录中.

$ cuz build

测试 #

运行测试命令将自动遍历并运行 test 目录下所有单元测试.

$ cuz test

指定环境变量 #

在默认情况下, 本地调试为开发环境(development), 构建项目为生产环境(production). 不过 cuz-cli 也支持您显示的声明当前环境变量, 具体方式如下:

$ cuz -e production
$ cuz build -e development
$ cuz --env custom-env  # 用户自定义环境
